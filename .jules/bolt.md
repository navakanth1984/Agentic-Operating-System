## 2024-05-24 - React Component Heartbeat Render Loop Optimization
**Learning:** High-frequency `setInterval` loops triggering `setState` (like the 1s telemetry graph heartbeat in `SystemResourcesView.tsx`) cause aggressive re-renders of the entire component. When combined with inline non-memoized array reductions, inline functional components (like `CustomTooltip`), and lack of lazy state initialization, this creates a massive amount of garbage collection churn and DOM diffing overhead every single second, visibly lagging a performance-heavy interface. Furthermore, multiple `.filter()` calls inside the main `App.tsx` render path compound this issue since global state updates trickle down.
**Action:** When designing or optimizing realtime dashboards, heavily utilize `React.useMemo` for any derived array statistics (`avgCpu`, `maxCpu`, filtered states), extract inline components out of the render body, and use lazy initialization `useState(() => initFn())` when default states are complex data structures.
